# AWSデータベース基盤構築手順 (Step 3)

本ドキュメントは、タスク管理システムにおけるAWS上のデータベース基盤（Amazon RDS）の手動構築手順を記録したものです。将来的なIaC（Terraform/CloudFormation）化の際の要件定義としても機能します。

## 目的

プライベートサブネット内にMySQLインスタンスを配置し、EC2（アプリケーションサーバー）からの通信のみを許可する最小権限のセキュリティグループを設定することで、セキュアなデータベース環境を構築します。

## 1. セキュリティグループの作成

データベースへのアクセスを特定のEC2インスタンスからのみに限定するため、2つのセキュリティグループを作成しました。

### 1.1 EC2用セキュリティグループ (送信元識別用)
- **名前:** todo-app-ec2-sg
- **説明:** Security group for Todo App EC2 instances
- **VPC:** todo-app-vpc
- **インバウンドルール:** なし（この段階では空の状態）
- **アウトバウンドルール:** すべてのトラフィック許可 (0.0.0.0/0)

### 1.2 RDS用セキュリティグループ
- **名前:** todo-app-rds-sg
- **説明:** Security group for Todo App RDS MySQL
- **VPC:** todo-app-vpc
- **インバウンドルール:**
  - **タイプ:** MySQL/Aurora (TCP / ポート 3306)
  - **ソース:** カスタム設定し、`todo-app-ec2-sg`のセキュリティグループIDを指定（これにより、EC2からのアクセスのみを許可）
- **アウトバウンドルール:** すべてのトラフィック許可 (0.0.0.0/0)

## 2. DBサブネットグループの作成

マルチAZ構成を想定し、RDSを配置するプライベートサブネットをグループ化しました。

- **名前:** todo-app-db-subnet-group
- **説明:** Subnet group for Todo App RDS
- **VPC:** todo-app-vpc
- **サブネット:**
  - ap-northeast-1a: 10.0.21.0/24 (todo-app-private-db-1a)
  - ap-northeast-1c: 10.0.22.0/24 (todo-app-private-db-1c)

## 3. Amazon RDS インスタンスの構築

リレーショナルデータベースとしてMySQLをプロビジョニングしました。アプリケーション側のコード改修や不要なランニングコストを防ぐため、シンプルなパスワード認証方式を採用しています。

### 基本設定
- **作成方法:** フル設定
- **エンジンのオプション:** MySQL
- **エンジンのバージョン:** 8.4.8 (ローカル環境と統一)
- **テンプレート:** 無料利用枠
- **DB インスタンス識別子:** todo-app-db

### 認証情報
- **認証管理:** セルフマネージド
- **マスターユーザー名:** admin
- **データベース認証オプション:** パスワード認証

### インスタンス構成
- **DB インスタンスクラス:** db.t4g.micro
- **ストレージタイプ:** 汎用 SSD (gp2)

### 接続設定
- **コンピューティングリソース:** EC2コンピューティングリソースに接続しない
- **ネットワークタイプ:** IPv4
- **Virtual Private Cloud (VPC):** todo-app-vpc
- **DB サブネットグループ:** todo-app-db-subnet-group
- **パブリックアクセス:** なし
- **VPC セキュリティグループ:** 既存の選択 (todo-app-rds-sg のみをアタッチ)

### モニタリング設定（初期設定のまま）
- **データベースインサイト:** スタンダード
- **拡張モニタリングの有効化:** オフ
- **ログのエクスポート:** すべてオフ

### 追加設定
- **最初のデータベース名:** todo_db (アプリケーション起動時の接続先として自動作成)
- **自動バックアップを有効にする:** オフ (学習用途のコスト削減・容量節約のため)
- **暗号を有効化:** オン (デフォルトKMSキー使用)
- **削除保護の有効化:** オフ (検証後のクリーンアップを容易にするため)

## 4. 構築完了後の確認事項

データベースのステータスが「利用可能 (Available)」になった後、詳細画面の「接続とセキュリティ」タブから以下の情報を取得し、環境変数設定用の秘匿情報として管理します。

- **エンドポイント:** `todo-app-db.xxxxx.ap-northeast-1.rds.amazonaws.com` (形式例)
- **ポート番号:** 3306

このエンドポイントと、作成時に指定したマスターユーザー名、パスワードは、後続のEC2構築ステップにおいて `.env` ファイルに設定します。
