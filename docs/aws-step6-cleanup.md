# AWS環境のリソース削除手順 (Step 6)

本ドキュメントは、タスク管理システムにおけるAWS上の構築済みリソースを手動で完全に削除し、不要な継続課金を防ぐためのクリーンアップ手順を記録したものです。リソース間の依存関係による削除エラーを防ぐため、構築と逆の順序で実行します。

## 1. ロードバランサーの削除

トラフィックの入り口となるロードバランサーと、その転送先設定を削除します。

- **ALBの削除:** EC2ダッシュボードから `todo-app-alb` を選択し、削除を実行します。
- **ターゲットグループの削除:** ALBの削除完了後、`todo-app-tg` を削除します。

## 2. EC2インスタンスとNAT Gatewayの削除

課金単価の高いNAT Gatewayの削除と、付随するElastic IPの解放を確実に行います。

- **EC2インスタンスの終了:** `todo-app-web-server` を選択し、「インスタンスを終了 (Terminate)」を実行します。
- **NAT Gatewayの削除:** VPCダッシュボードから `todo-app-nat-gw` を削除します。ステータスが「Deleted」になるまで数分待機します。
- **Elastic IP (EIP) の解放:** NAT Gatewayの削除完了後、割り当てられていたEIPを必ず「Elastic IP アドレスの解放」から手放します（解放漏れは継続課金の原因となります）。

## 3. データベース (RDS) の削除

- **RDSインスタンスの削除:** RDSダッシュボードから `todo-app-db` を削除します。最終スナップショットの作成は「不要」とし、自動バックアップも保持せずに削除します。
- **DBサブネットグループの削除:** インスタンス削除完了後、`todo-app-db-subnet-group` を削除します。

## 4. セキュリティグループの削除

対象のリソース（EC2, RDS, ALB）がすべて削除されたことを確認した後、セキュリティグループを削除します。

- `todo-app-alb-sg`
- `todo-app-rds-sg`
- `todo-app-ec2-sg`

※相互にルールを参照している場合は、先にインバウンド/アウトバウンドルールを削除してからグループ自体を削除します。

## 5. ネットワーク基盤 (VPC) の削除

最後にネットワークの枠組みを削除します。

- **VPCの削除:** VPCダッシュボードから `todo-app-vpc` を削除します。通常、これに伴い関連するサブネット（6つ）、ルートテーブル、インターネットゲートウェイ（`todo-app-igw`）も自動的に削除されます。

## 6. IAMロールの削除 (任意)

- **ロールの削除:** IAMダッシュボードから `todo-app-ec2-ssm-role` を削除します。

## 7. 最終精査 (消し残りの確認)

すべての削除作業が完了した翌日以降に、以下のツールを使用してリソースの消し残りや予期せぬ課金が発生していないか厳しく精査します。

- **AWS Resource Explorer:** `todo-app` で全体検索を実行。
- **Tag Editor:** タグのキー `Name` に `todo-app` を含むリソースを検索。
- **Cost Explorer / Billing Dashboard:** 数日後のコスト推移を確認し、EIPやKMS、スナップショットなどの微細な課金が継続していないか確認。
